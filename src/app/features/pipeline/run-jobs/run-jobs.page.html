Dashboard Trend Run Configuration

This job triggers memory-based niche recommendations for the selected target audience. It uses the user's active niche profile stored in core memories and calls the LLM provider to generate a three-tier JSON list (Today, Weekly, Monthly) of trending keyword recommendations. The results are stored directly into the database to be displayed on the User's Dashboard.

Execution Strategy: Light-weight background job
Local Execution: True (Admin Local Queue)
Processing Run Configuration

Triggers a raw folder scan for target users. The scanner checks the user's Google Drive raw video folder for unprocessed files, implements a distributed file name locking protocol (claiming the files as WORKING), inserts job records in the database, and schedules video pipeline tasks. The actual heavy rendering tasks are enqueued to standard Hangfire processing queues and routed to the Backend server.

Execution Strategy: Distributed scan & heavy background rendering
Hangfire Queue: processing
Trend Run Configuration

Triggers a heavy social trend cycle. This will fetch real-time external trend inputs, execute the niche mapping model, rank relevance scores against the user profile settings, and enqueue social media post generation pipelines. The heavy workflow tasks are routed to the Backend Hangfire worker nodes on the trends queue.

Execution Strategy: Heavy external trend discovery & social scheduling
Hangfire Queue: trends
